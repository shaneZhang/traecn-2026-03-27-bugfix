## 关键 Bug 修复验证

### 1. URL 查询参数编码问题 ✅

**修复前:** `?` 被编码为 `%3F`，导致 API 返回 404
**修复后:** 查询参数正确传递

**验证命令:**
```bash
./mastodon-cli follow zhangyuqing
# 结果: You are now following @zhangyuqing ✅
```

### 2. visibility 参数被忽略 ✅

**修复前:** 无论指定什么 visibility，帖子都是 public
**修复后:** visibility 参数正确传递

**验证命令:**
```bash
./mastodon-cli post "Test private" -v private
# 结果: 帖子以 private 可见性发布 ✅
```

### 3. hasScheme 越界错误 ✅

**修复前:** 长度为 7-8 的 URL 会导致 panic
**修复后:** 正确处理各种长度的 URL

**验证:** 所有 API 调用正常工作，无 panic

### 4. isID 函数误判 ✅

**修复前:** 用户名如 "123" 会被误判为 ID
**修复后:** 只有 18 位以上的数字才被识别为 ID

**验证命令:**
```bash
./mastodon-cli account 116287289073314595  # 18位ID，使用ID查询 ✅
./mastodon-cli account zhangyuqing          # 用户名，使用用户名查询 ✅
```

### 5. Search API 版本错误 ✅

**修复前:** 使用 v1 search API，某些实例返回 404
**修复后:** 使用 v2 search API

**验证命令:**
```bash
./mastodon-cli search "zhangyuqing"
# 结果: 找到用户 @zhangyuqing ✅
```

### 6. in_reply_to_id 参数名错误 ✅

**修复前:** 使用 `in_reply_to`，回复无法关联
**修复后:** 使用 `in_reply_to_id`

**验证命令:**
```bash
./mastodon-cli reply <status_id> "Test reply"
# 结果: Reply posted! ✅
```
